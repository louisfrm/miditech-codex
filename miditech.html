Après la tâche 07

Sécurité

Comprenez ce que Codex peut faire par défaut et quand il doit vous demander une approbation.

  1. Deux couches de sécurité : le sandbox limite techniquement l'accès, la politique d'approbation détermine quand Codex doit demander votre feu vert.
  2. IDE uniquement : les permissions sont pilotées par le mode sélectionné :
    • Chat : discussion et planification sans modifications automatiques.
    • Agent : modifications possibles dans le workspace, avec approbations pour écrire en dehors du workspace ou accéder au réseau (sauf si network_access = true est activé dans la config).
    • Agent (full access) : désactive la sandbox et autorise les actions sensibles sans demander d'approbation (ex : accès réseau, lecture/écriture étendue).
  3. Web search : options possibles :
    • "disabled" pour désactiver l'outil.
    • "cached" (par défaut) pour utiliser un index mis en cache.
    • "live" pour interroger le web en direct.

Référence : guide sécurité officiel Codex.

[sandbox_workspace_write]
network_access = true

web_search = "cached" # default
# web_search = "live"
# web_search = "disabled"
Lire le guide sécurité

Avant la tâche 09

Qu'est-ce que le MCP ?

Le Model Context Protocol (MCP) est un standard open-source qui connecte les applications d'IA à des systèmes externes (données, outils et workflows).

Pensez au MCP comme à un port USB-C pour l'IA : un connecteur standard qui permet de brancher facilement des sources et des capacités.

  1. Ce que ça permet : accéder à vos fichiers, bases de données, outils et services.
  2. Pourquoi c'est utile : moins d'intégration sur-mesure pour les devs, plus de capacités pour les apps, meilleure expérience pour les utilisateurs.
  3. Exemples : assistants reliés à Calendar/Notion, bots d'entreprise multi-bases, génération d'apps depuis Figma.

Source : documentation "Getting Started" du MCP.

Lire l'intro MCP

Tâche 11

Configurer le serveur MCP GitHub

Ajoutez le serveur MCP GitHub (hébergé par GitHub) afin que Codex puisse interagir avec vos repos.

  1. Créez un Personal Access Token GitHub avec les scopes minimum nécessaires via https://github.com/settings/personal-access-tokens/new, puis stockez-le dans une variable d'environnement nommée GITHUB_PAT_TOKEN.
  2. Si vous utilisez un fichier .env local, ajoutez-le à .gitignore pour éviter toute fuite.
  3. Dans l'IDE, ouvrez les réglages Codex puis le fichier config.toml (commun CLI + IDE).
  4. Ajoutez la section [mcp_servers.github] avec l'URL du serveur distant.
  5. Enregistrez, relancez Codex si besoin, puis utilisez /mcp ou le panneau MCP pour vérifier que github apparaît.
[mcp_servers.github]
url = "https://api.githubcopilot.com/mcp/"
bearer_token_env_var = "GITHUB_PAT_TOKEN"

Exemple de stockage local (évitez de committer ce fichier) :

GITHUB_PAT_TOKEN=ghp_votre_token_ici
